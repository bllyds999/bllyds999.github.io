最近，我把日常开发任务的主力工具切换到了 Claude Code。有人可能会疑惑：你不是一直在用 DeepSeek TUI 吗？怎么突然换了？其实这只是一次“套壳”尝鲜——我依然使用 DeepSeek 的 API 作为后端，只是前端换成了 Claude Code 这个终端交互工具。网上有非常详细的切换教程，跟着配置环境变量和 API 地址就能轻松搞定。体验下来，不得不承认 Claude Code 作为终端工具的完成度确实远超目前的开源方案，无论是响应速度、上下文管理还是交互流畅度，都给人一种“大厂出品”的扎实感。虽然 Anthropic 这家公司并非由国人创办，但它们打造的产品质量确实优秀。把这个壳套在 DeepSeek 的 API 上使用，竟然异常顺手，效率提升明显。至于注册 Claude 账号？我压根就没动过这个念头，毕竟我只是需要一个好用的终端界面，而 DeepSeek 的低价 API 才是真正的算力来源。

今天是 7 月 2 日，我像往常一样刷着技术视频，原本以为这只是一个普通用户在吐槽 A 社封禁账号的日常抱怨。结果点开一看，这位老哥直接放了个大招——他花了大量时间爬取了全网关于 Claude Code 检测机制的研究资料，从 IP 归属地到城市级别的定位，把 Anthropic 如何识别中国用户的技术手段扒了个底朝天。最让我后背发凉的是视频发布日期：不是上个月的老视频，恰恰就是 7 月 1 日，几乎就是昨天才发布的内容。那一刻我坐在电脑前，感觉自己的底裤都快被扒光了。首先，Claude Code 会读取你系统的时区代码，精准判断你身处哪个国家；其次，它会在你毫不知情的情况下对提示词做“一点点”修改，比如把 `2026-07-02` 这种标准格式，根据你的时区自动改写为 `2026/07/02` 这样的本地化格式——这种细微的调整普通用户根本不会注意到。

更隐蔽的操作还在后面。它居然会在引号上做手脚——给你的提示词里塞进四种不同类型的引号，正常用户的文本会显示为普通 ASCII 引号，而被标记为“异常”的用户则会看到其他 Unicode 变体引号，你只会天真地以为是字体渲染差异，根本没往检测手段那方面想。更令人发指的是，它还在你的申诉邮件中埋入 JavaScript 代码，用来获取你的真实 IP 地址进行二次验证。一个号称只是用来辅助编程的命令行工具，内部竟然内置了一长串域名黑名单，像百度、阿里这些国内最基本的大厂全被列了进去，唯独腾讯不在其中。有网友分析认为这根本不是黑名单，而是正版验证机制，因为名单上出现的中转域名确实会真的向 Anthropic 的服务器发起请求——这意味着它每时每刻都在验证你的“身份合法性”。

我平时在自己的网站 Umami 分析后台，都能通过 IP 看到访客来自哪个城市，甚至能猜到是哪位朋友来看我的博客。现在想想，Claude Code 拿了你那么多信息——时区、IP、系统语言、提示词格式、引号习惯、邮件行为——居然仅仅只是为了判断你是不是“合规用户”，然后给你封号处理。可万一哪天这家公司不仅仅满足于封号呢？这些数据会流向哪里？会被用来做什么？越想越觉得脊背发凉。虽然我从未注册过 Claude 账号，全程用的只是最便宜的 DeepSeek API，理论上和 Anthropic 没有任何直接关系，但那种被暗中监控的感觉依然让人极度不适。思来想去，我那些日常任务用 DeepSeek TUI 或者 iFlow 其实完全够用，何必为了那一点交互体验的提升而把自己的隐私暴露给一个不可控的闭源工具？于是我毫不犹豫地执行了卸载命令：

```shell
npm uninstall -g @anthropic-ai/claude-code
rm -rf ~/.claude
rm -rf ~/.cache/claude-code
```

如果你也在使用 Claude Code 并且对隐私问题有所顾虑，可以按照上面的命令彻底清理干净。至于替代方案，Codex 倒是开源的，但它是 OpenAI 旗下的项目，同样是另一个科技巨头的产物，除非你对 OpenAI 的信任度更高，否则换汤不换药。剩下的选择就是 DeepSeek TUI、OpenCode 这些纯粹的开源社区产品了，它们托管在 GitHub 仓库中，代码完全公开透明，没有任何埋点或检测逻辑——虽然功能上可能没那么花哨，但至少用着安心。毕竟在如今这个时代，隐私比那一点效率提升珍贵太多了。
