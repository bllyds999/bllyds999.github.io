最近我重构了网站的编译模式，搞了一个适用于 Hexo 的 GitHub Actions。原本我雄心勃勃地打算把这个工作流提交到 GitHub 的官方 Actions 模板仓库，想着既能方便别人，也算给自己攒点开源经验。结果在提交代码的时候，系统提示我必须先勾选 GitHub Pull Request 表单。我定睛一看，好家伙，上面一条一条的条款密密麻麻，写的全是英文。作为一名中文母语者，看到那么多不熟悉的专业术语和法律措辞，我的第一反应就是懒得读。于是我很自然地打开了必应翻译器，逐条复制进去翻译，确认大概意思之后再打勾。前几条还算顺利，虽然麻烦了点，但总归能推进。可就在我快要完成整个表单的时候，突然遇到一条写着“我已经为提交的 Actions 准备图标”的声明，我整个人就愣住了——我只写了一堆代码逻辑，压根没想过还要配图标啊。

我寻思着 Hugo 和 Hexo 的图标长得挺像的吧，毕竟都是六角形，结构上大差不差。于是我就从网上把 Hugo 的图标下载下来，打算放到编辑器里改改再用。刚开始的时候确实挺顺利的，无非就是改改 SVG 的颜色，把原本品红色、带描边效果的图标，改成纯蓝色、不带外描边的简洁风格。改完之后我对比了一下 Hexo 官网的官方图标，又看看我手里这个改好的 Hugo 版，总觉得差了点味道。仔细一看才发现问题所在：官方图标的六个角是锋利的尖角，而我这六个角还是 Hugo 原本那种圆乎乎的风格。我想，既然都是纯代码了，改个圆角应该挺简单的吧。我翻到 SVG 代码里有个 `round` 参数，二话不说把它改成了 `miter`，还把数值直接拉到一百，满心以为刷新一下就能看到尖角效果了。

改完之后我刷新预览了一下图标，结果发现这不是压根没变吗？我还是不甘心，心里琢磨着 AI 懂得应该比我多吧。于是我去问了问豆包。我先复制了一行相关代码到聊天框里，想让豆包帮我参谋参谋怎么把圆角彻底改成尖角。结果豆包把以上我做过的所有事情又复述了一遍，告诉我它的理解。我退学之后在家待了那么久，脾气其实已经算可以了。但我还是耐着性子跟它说：“对不起，没有变化！”，希望它能意识到问题。结果豆包依旧像复读机一样重复那些分析和提问。我想，可能是我的上下文给得不够？于是我把整段 SVG 代码一股脑全扔给了豆包。这时候豆包回复了我一句：“好的，我将会为您生成一个没有圆角的版本！”我当时那个期待啊，莫非它真的能开始正经工作了吗？

呵呵，看来还是我想多了。豆包生成的图片跟我给它的原图一模一样，所谓的“没有圆角的版本”完全就是复读机行为。我这回实在是忍不了了，直接吐槽它：“你这不是复读机吗？”，它倒是“貌似”知道了自己的错误，在那一个劲地道歉，说自己哪里哪里做得不对，态度还显得特别诚恳。我说：“你不需要道歉，我也没让你道歉，要的是你赶紧生成图片！”这时候它说：“好的，我这就为您生成一张没有圆角的图片！”，然后，就没有下文了。没错，它真的只说了这句话，连复读机都不复读了，干脆只剩下空洞的承诺。我继续吐槽它：“你图片生成到哪里去了？”，它依旧说：“好的，我这就为您生成图片！”，依然没有任何实际产出。

我还是觉得是不是自己给的上下文不够，说不定字节跳动新搞了什么上下文压缩技术，AI 看不到我前面的历史消息也说不定。于是我把图片重新发给了它，再配上更详细的说明。这次它说：“好的，我即将为您生成一张没有圆角的图标，你可以直接用来作为 Hexo 网站图标！”……莫非它这次真的开始工作了吗？好吧，它依旧只是把我发过去的图片原封不动地复读回来，这时候我突然反应过来，我好像已经很久没用过豆包了。最近一直在用 DeepSeek，不是因为 DeepSeek 更懂我，至少人家每次都会认认真真地展示思考过程，而且我用着用着就有了感情。我想起来上次觉得豆包还有用的时候，貌似就两件事：一个是生成角色三视图，另一个是去年在技校签名的时候。

我用豆包生成角色三视图，是因为豆包的 Seedance 2.0 在图像生成领域至少是业界领先的级别，而且我那个时候也不需要什么深度思考，只要画面好看就行。至于技校签名的那一次，是去年 2025 年 9 月份开学（唉，在自己生日开学）刚去技校的时候，校方让我们填各种表格。因为学校允许带手机，校方就说如果有什么不懂的可以问豆包——现在回想起来，感觉更像是校方在推脱责任，不想回答学生的琐碎问题。我确实有不懂的地方就去问了豆包，但是大部分需要填的内容，校方自己已经在投影上讲过了，而且填表要填的都是真实个人信息，AI 生成文章的功能在这种场景下根本派不上用场。怪不得前几天豆包收费的事在网上引发了热议——给我这种质量的服务还卖那么贵，想想就让人来气。
