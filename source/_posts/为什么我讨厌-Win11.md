我曾是 Windows 的忠实用户，因为我是从 Win7 开始接触 Windows 的，那是在我爸的电脑上；后来我有了第一台属于自己的电脑，预装的是 Win10 系统。这两个系统在外界的评价都相当好，也是我认为的电脑应该有的样子——它至少应该是个工具，要有功能性、稳定性，不能是儿戏。Win7 在当时被誉为最经典的版本之一，其界面简洁明了，任务栏、开始菜单的布局让新手也能快速上手。而 Win10 则在此基础上融合了现代触控元素，同时保留了传统桌面的高效操作逻辑。对我来说，一个操作系统最重要的不是花哨的动画或莫名其妙的交互创新，而是每次点击都能得到预期的响应，长时间运行不卡顿、不蓝屏。Win7 和 Win10 恰好都满足了这些要求，所以在我心中，它们定义了“电脑该有的样子”——一台可靠的生产力工具，而不是需要用户不断去适应和妥协的半成品。

接触 Win11 的契机是 2022 年那次系统更新，当时我从社交媒体上了解到了这个全新的、看起来貌似很现代化的系统。作为忠实用户，抱着尝鲜的目的，我从测试通道更新了 Win11。刚开始更新的时候，确实觉得漂亮，和我以前见过的 Win10 根本不是一个风格的，这种全新的风格叫阳光谷。社交媒体上铺天盖地的截图和视频都在展示 Win11 的全新设计：居中的开始菜单、圆角窗口、毛玻璃特效，以及重新设计的图标和设置界面。我怀着兴奋的心情加入 Windows 预览体验计划，下载并安装了第一个测试版本。开机动画变了，登录界面的字体和布局也变了，整个系统仿佛披上了一件时尚的外衣。那种新鲜感让我觉得自己走在了科技潮流的前沿，毕竟当时大多数用户还在使用 Win10，而我已经提前体验到了微软所谓的“下一代 Windows”。

我用 Win11 蛮长时间的了，刚升上来并没有遇到那些人在社交媒体上吐槽的“开始按钮位置变化”问题。这不是微软还没有修改，完全只是因为当时我还在学编程，脑子灵光，知道可以从设置里面修改。很明显，这时候我用的 Win11 和后来的差远了，完全只是 Win10 的皮肤。那时候的 Win11 底层依然是 Win10 的核心，驱动程序、兼容性、系统服务都没有本质变化，主要的改动集中在用户界面层面。开始菜单从左侧移到了中间，但通过任务栏设置可以轻松改回左对齐；右键菜单变成了折叠式，但点一下“显示更多选项”就能回到传统样式。这些浅层修改并没有触及系统内核，所以运行原有的软件和游戏都没有遇到太大的兼容性问题。对我而言，这就像给 Win10 换了一套主题皮肤，新鲜但不震撼，稳定但也没有质的飞跃。

当时的 Win11 作为 Win10 的皮肤，虽然外表上看差别挺大，但是用起来至少还是能用的，因为只是 Win10 的皮肤。令人感到厌烦的就是我在 Win11 上和同学打原神（当时初一是 2022 年，还是疫情停课的时候），那台联想上网本烂电脑，反反复复地死机，这给了我 Win11 一点也不稳定的印象。那台上网本的配置本来就低：赛扬处理器、4GB 内存、机械硬盘，Win10 运行起来已经捉襟见肘，装上 Win11 更是雪上加霜。每次启动原神，风扇都会狂转，画面时不时卡顿，最让人崩溃的是在打 boss 的关键时刻突然蓝屏或直接黑屏重启。同学在语音里抱怨我掉线，我只能苦笑着重启电脑，重新加载游戏。这种情况每周都会发生好几次，我开始怀疑 Win11 对低端硬件的优化是否彻底失败。微软官方宣称 Win11 对硬件要求更高，而我的亲身体验恰恰证实了这一点：不是 Win11 本身不稳定，而是它不适合在老旧的入门级设备上运行。

后来换了一台游戏本（现在那台放闲鱼卖掉了），虽然系统预装的是 Win11，但那时候的 Win11 已经相对稳定了（毕竟是 2024 年换的电脑）。我先是在游戏本上体验了以前在上网本没玩过的光追，又搞了个 XGP 玩刺客信条，我依旧记得那种开着空调躲在卧室里，游戏里还在沙漠的感觉。新电脑配备的是 RTX 40 系列显卡、16GB 内存和高速固态硬盘，Win11 在上面跑得丝般顺滑。我终于打开了以前想都不敢想的光线追踪特效，在《赛博朋克 2077》里，霓虹灯的倒影在水洼中清晰可见；通过 XGP 订阅服务，我下载了《刺客信条：起源》，置身于古埃及的沙漠中，烈日、沙尘、金字塔都栩栩如生。那种沉浸感令人难忘：卧室里空调吹着冷风，窗外是夏天，而游戏角色却在炙热的沙漠里奔跑，皮肤仿佛能感受到屏幕里的热浪。Win11 在这个过程中表现得很稳定，没有一次崩溃或卡顿，我意识到问题不在于系统，而在于硬件配置。

因为有顾虑之忧，我的系统从 Win11 换到了 Linux。换到 Linux 后，我经常关注一些开源新闻。很多开源新闻创作者，尤其是 Linux 的开源新闻创作者，就特别喜欢吐槽 Win11，加上当时 Win11 确实在社交媒体上引起连连哀嚎，这让我忽视了 Linux 的图形界面究竟有多么难用，还在庆幸自己远离了 Windows 的魔爪。我每天刷着各种科技论坛和视频频道，看到博主们列出 Win11 的各种槽点：强制合并任务栏图标、右键菜单需要二次点击、开始菜单推荐项无法彻底关闭、系统设置和控制面板依然割裂……这些抱怨让我相信 Win11 是一个失败的作品。与此同时，Linux 社区的声音清一色在夸赞开源系统的自由、高效和可定制性，仿佛只要切换到 Linux，所有问题都会迎刃而解。我被这些言论感染，忽略了 Linux 桌面环境对于普通用户的学习成本和日常使用中的各种小麻烦，天真地以为自己找到了完美的替代品。

经过一段时间之后，我实在忍不了 Linux 这个图形界面了。GNOME 和 KDE 不折腾一下都太难用了，平铺桌面用起来虽然方便，但是总感觉差点什么。我想装回去 Windows，可是我只会装 Linux 系统啊，我不知道怎么装回。好在后来我学会了装回 Windows，我选择了 Win10 作为装回去的系统。GNOME 的默认布局过于极简，连桌面图标都不显示，最小化和最大化按钮默认隐藏，每次安装软件都要去找“显示应用程序”菜单；KDE 虽然可定制性强，但设置选项多得让人眼花缭乱，调整一个主题可能要翻几十个菜单项。我尝试了 Hyprland 和 Niri 这样的平铺窗口管理器，用键盘控制窗口布局确实高效，但运行图形化的日常软件（比如浏览器、截图工具、输入法配置）时总会遇到各种兼容性问题。更让我崩溃的是，每次系统更新后，显卡驱动或 Wi-Fi 驱动都可能出问题，需要花半天时间查文档修 BUG。折腾了两个月后，我终于下定决心回到 Windows，可当时我只会用 dd 命令烧录 Linux 启动盘，完全没想过 Windows 也有安装介质。后来借了同学的电脑，从微软官网下载了媒体创建工具，才成功装回了 Win10。

此刻我完全走到了另一个极端，认为 Linux 图形界面就是笑话，完全不如 Windows 的图形界面。当然，人教人不会，事教人一遍对，很快我就从社交媒体上了解到 Win10 即将停止更新的事情，大概是去年十月份（2025 年 10 月）吧！你当然可以更新，但是你需要给比尔盖茨叔叔递钱钱才能继续更新。刚装回 Win10 的那几天，我感觉像回了家一样亲切：熟悉的开始菜单在左下角，右键菜单直接显示全部选项，任务栏可以随意拖动位置，所有设置都在一个统一的控制面板里。我甚至嘲笑自己之前为什么要在 Linux 上浪费那么多时间，Windows 明明把所有事情都做得简单直接。然而这份得意没有持续太久，各大科技媒体开始密集报道 Win10 生命周期终止的消息 —— 2025 年 10 月之后，微软将停止向 Win10 提供免费的 安全更新。普通用户可以继续使用，但如果想获得扩展安全更新，就必须按设备支付年费，这被网友戏称为“给比尔盖茨叔叔递钱钱”。这条消息像一盆冷水浇下来，让我又一次陷入了焦虑。

当时我心里又想，我用 Win10 就是没未来，我用 Linux 就是有未来但没现在。于是我开启了一段 Distro Hopper 时光，换到 Linux 时怀念 Windows 多么多么好用，换到 Win10 时想到 Linux 有而 Windows 没有未来……这种症状一直持续了很久，直到我卖掉了那台游戏本，买了台 Macbook Air。那段时间我像是着了魔一样，每周都在重装系统：周一换成 Gentoo，周三因为 Wi-Fi 驱动问题换回 Win10，周五又看到新闻说 Linux 某个发行版对游戏支持不错，于是再次格盘安装。每次切换都是一场精神内耗——用 Linux 时打开 Office 文档排版会乱，截图工具不好用，网易云音乐客户端时不时闪退；用 Win10 时又开始担心 2025 年之后的安全补丁问题，害怕自己的数据会被新出现的漏洞威胁。直到有一天，我看着满健康度差到离谱的硬盘，突然觉得无比疲惫。我卖掉了那台游戏本，在苹果官网买了一台 MacBook Air，心想：也许换个完全不同的生态，就能结束这种无休止的折腾了。

我感觉微软其实知道如何做一个操作系统，因为我作为一个用户，我完全知道现在所谓的高级感 UI，无非就是同模换色。用户渴望的，仅仅只是能在笔记本上安静地追剧、工作。你只需要把系统打磨到让这个时代的用户觉得满意，而不是总想着换各种皮肤定义下一个时代，就完美了。如果能做到少更新、多安全维护，那简直是最棒的团队了。经历了这么多系统的来回切换，我逐渐明白一个道理：用户要的不是每天都有新花样，而是每次打开电脑都能获得稳定、一致的体验。Win10 的深色模式、任务栏、文件资源管理器，哪怕界面不够“现代”，但只要用熟了就能形成肌肉记忆。微软这些年却热衷于在 UI 上玩花样：Win11 把开始菜单从左边移到中间，改了右键菜单，刷新了设置图标，但这些改动并没有带来实质性的效率提升，反而让老用户需要重新学习。我想要的是一个可以安心写文档、追剧、偶尔玩玩游戏的操作系统，不希望每个月的大更新都提心吊胆，怕某个驱动炸了或者打印机连不上了。少而精的更新、长期的安全维护，远比“一年两更、每更半车 BUG”要让人舒心。

我了解到，Windows 部门以及所谓的微软测试团队，早就被砍掉了。Windows 部门的业务全部交由 Surface 部门接手，用户则承担了微软没有测试部门的错误，为微软当免费工具人。一个做单一品牌电脑的，去负责一个给所有品牌做通用系统的业务，本身就是个玩笑。先不说理念高不高级，最起码第一步就搞错了。过去微软有一个庞大的 Windows 测试团队，他们在新版本发布前会在数千种不同的硬件配置上进行兼容性测试。然而几年前，微软裁掉了这个测试团队，转而让 Windows 部门并入 Surface 部门，同时推行“Windows 即服务”模式，意思是让 Insider 用户来充当免费测试员。这意味着微软不再主动在各种品牌和型号的设备上充分验证更新，而是先把补丁推送给预览体验成员，等他们反馈了 BUG 再慢慢修复。更离谱的是，Surface 部门只专注于微软自家那几款硬件（Surface Laptop、Surface Pro 等，但是它们早期都被人骂惨了），而 Windows 本应运行在联想、戴尔、惠普、华硕等成百上千种不同配置的电脑上。用一个单一硬件团队的思路去管理一个通用操作系统，这从一开始就是错的。用户遇到蓝屏、驱动不兼容、打印机无法识别时，微软却指望我们去反馈中心写报告，这不是专业，而是把责任转嫁给了消费者。
