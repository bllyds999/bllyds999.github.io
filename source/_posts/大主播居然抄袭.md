我在 B 站漫无目的地刷着 MC 视频，算法突然给我推了一个从未见过的实况主。点进去一看，画面精致，光影柔和，方块纹理细腻得像是重新定义了游戏的美学。我正打算投币，弹幕里却炸开了锅——“这不就是阿普修的材质包吗？”、“连草方块的侧面纹理都照搬？”、“UP主好意思写原创？”我一脸疑惑，心想这内容不是挺用心的吗？是不是观众太敏感了。直到有人甩出一条链接，点进去赫然写着“阿普修·真实系材质包 V3.2”，预览图里的石头纹理、水纹折射、甚至铁轨的锈迹角度，都和这个视频里如出一辙。我愣住了，原来这不是普通的致敬，而是近乎像素级的“撞车”。

评论区很快变成了一场混乱的狂欢。有人耐心地截出两组图片，把视频里泥土方块边缘的暗纹和阿普修的材质包并列，用红圈标出完全相同的锯齿形状；有人晒出阿普修材质包里一个著名的渲染 BUG——某个特定角度下玻璃板会出现一条微小的紫色亮线，而视频里也恰好在相同位置出现了同样的问题；还有人干脆跑题，发自己的猫、刚做的饭、甚至一张搞怪自拍，说“我也来蹭个热度”。楼中楼里骂声和笑声交织，有维护博主的，有嘲讽的，有吃瓜求总结的，整个评论区乌烟瘴气，群魔乱舞。我在这一片混乱中显得格外茫然，根本拼凑不出完整的事件全貌。这时一个好心人私信我：“看你好像是阿普修的朋友，要不你去跟他说一声？”我犹豫了几秒，还是点开了阿普修的私信窗口。

我花了点时间把视频链接和评论区里看起来最实锤的证据一起私信给了阿普修。过了不到半小时，他回复了，消息里带着明显压抑不住的怒意：“就是我的纹理，有几块是我手绘时特意留下的不对称结构，他连那个都没改。”接着他又发了一组更为专业的数据对比——两个材包的元数据生成时间戳、PSD 文件中未清理的图层名称、甚至某个云纹的噪点分布都几乎一致。阿普修更新了一条动态，直言“被抄得骨头都不剩”，并附上了自己的原始工程文件时间截图。至此，评论区迅速一边倒地谴责那位博主。我心里一块石头落地，觉得事情总算尘埃落定，真相大白，可喜可贺。我关掉页面，准备去睡觉，心想今天也算是见义勇为了一回。

然而第二天一早，我习惯性地点开 B 站，首页第一条推荐赫然又是这个事件——只不过标题变了，从“道歉回应”变成了“反转！谁抄谁还不好说”。我脑子嗡了一下，什么情况？不是说已经被实锤了吗？点进去一看，是那位博主刚刚发布的一则长视频，封面大字写着“自证清白”。评论区再次炸开，原本一边倒的舆论瞬间分裂成两派。我越看越懵，手心开始出汗——难道我昨天当了一回错误的传话筒？视频里，那位被指控的大主播没有多费口舌辩解，而是直接共享了自己的电脑屏幕。他打开了一个代码托管网站 GitHub 上的仓库，创建日期赫然显示在两年前——比阿普修材质包的首个公开发布版本还要早三个月。

他依次点开每一个纹理的 `.png` 文件，右侧信息栏里的“创建时间”和“修改时间”全部清晰可见。他还调出了自己本地的 Git 提交记录，一条一条地滚动展示：“2022 年 3 月 12 日，添加草方块侧面纹理；2022 年 5 月 8 日，修正石砖阴影……你们自己看，这些时间戳能造假吗？”他强调自己一直是个开源开发者，这个材质包从第一天起就放在 GitHub 上无人问津，只有零星几个星标。他没有合并过任何外部的 Pull Request，所有纹理都是自己逐个像素画出来的。直到最近终于把 500 多个方块全部画完，才鼓起勇气做了第一个宣传视频，没想到迎来的不是认可，而是“抄袭”的骂名。

他叹了口气：“我画了两年，连个 PR 都没敢接，就是怕混入别人的东西。结果你们说我是抄的？”
